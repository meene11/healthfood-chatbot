## 2. 평가 지표 설계

### 지표 1: 검색 커버리지 (Retrieval Coverage, 0~1)
- **방법**: source_content의 핵심 키워드(2글자 이상 명사/영어)가 검색된 문서에 포함되는 비율
- **의미**: 내 챗봇이 조원과 같은 소스 정보를 검색해냈는가

### 지표 2: 답변 품질 (Answer Quality, 0~3) — LLM-as-Judge
- **방법**: GPT-4o-mini가 생성 답변을 reference_answer와 비교해 0~3점 채점
- **기준**:
  - 3점: 완전 정답 (reference와 동등하거나 더 좋음)
  - 2점: 대부분 정확 (핵심은 맞으나 약간 불완전)
  - 1점: 부분 정답 (중요한 내용 누락)
  - 0점: 오답 또는 "모릅니다"

### 지표 3: 응답 시간 (Response Time)
- RAG 검색(임베딩 + Supabase + 리랭킹) + LLM 생성 전체 시간

### 낮은 커버리지의 이유

커버리지 0.215는 낮아 보이지만:
- 조원의 데이터는 **다른 PDF 파일 세트** 기반 (pdf-data/*.pdf)
- 내 DB(documents_v2)는 한국어 건강식품·다이어트 논문 중심
- 일부 질문은 영어 논문 참고문헌/약어 섹션 기반 → 내 DB에 없는 정보
- 반면 답변 품질 1.58/3.0은 **실질적인 답변 생성 능력**을 보여줌

## 4. 모델 비교 계획

### 비교할 모델 3개 + 현재 모델

| 순서 | 모델 | 제공사 | 비용 ($/1M 입력) | 특징 |
|:----:|------|--------|:---------------:|------|
| ① **현재** | GPT-4o-mini | OpenAI | $0.15 | 기준선. 빠르고 저렴 |
| ② **예정** | GPT-4o | OpenAI | $2.50 | 최고 추론력. 복잡한 의학 질문에 강함 |
| ③ **예정** | Claude Haiku 4.5 | Anthropic | $0.08 | 최빠름·최저가. 한국어 답변 품질 탁월 |
| ④ **예정** | Llama 3.3 70B | Meta/Groq | 무료 | 오픈소스. Groq 무료 API로 비용 없음 |

### 모델 선정 이유

**GPT-4o** — 같은 OpenAI API, API 키 추가 불필요. 비용이 17배 비싸지만 복잡한 의학 메커니즘 질문(mTOR, BCAA 대사 등)에서 얼마나 차이나는지 직접 비교 가능.

**Claude Haiku 4.5** — Anthropic API(별도 키 필요). GPT 계열과 다른 학습 방식. 한국어 자연스러움과 긴 문맥 이해력에서 차이를 확인. 비용은 가장 저렴.

**Llama 3.3 70B (Groq)** — 완전 무료 오픈소스. 상용 모델 대비 성능 격차를 수치로 확인. Groq API는 OpenAI 호환이라 코드 변경 최소.

### 비교 평가 실행 방법

```bash
# Anthropic API 키 .env에 추가 후:
python src/evaluate_teammate_qa.py --model gpt-4o

# Anthropic 계정 후:
python src/evaluate_teammate_qa.py --model claude-haiku

# Groq 무료 계정 후:
python src/evaluate_teammate_qa.py --model groq-llama
```

### 기대 가설

| 모델 | 예상 품질 | 예상 시간 | 비용 |
|------|:--------:|:--------:|:----:|
| GPT-4o-mini (현재) | 1.58/3.0 | 8초 | 낮음 |
| GPT-4o | 2.0~2.3/3.0 | 10초 | 높음 |
| Claude Haiku | 1.7~2.0/3.0 | 5초 | 가장 낮음 |
| Llama 3.3 70B | 1.3~1.6/3.0 | 3초 | 없음 |

### 비교 후 생성될 차트

- 모델별 평균 품질 비교 바 차트
- 모델별 완전정답/오답 분포 누적 바
- 비용 대비 성능 (cost-effectiveness) 산점도
- 카테고리별 모델 성능 히트맵
