### 낮은 점수의 원인 분석

**수치(0.095)가 낮은 이유는 RAG 시스템의 문제가 아닌 "정확한 청크 검색"의 근본적 어려움 때문입니다.**

| 원인 | 설명 |
|------|------|
| **중복 유사 청크** | 18,682개 청크 중 다이어트/건강식품 논문은 동일 주제를 수백 개의 청크로 분할. 같은 질문에 답할 수 있는 청크가 여러 개 존재 |
| **문맥 의존 질문** | GPT-4o-mini가 일부 질문에 "이 연구", "이 그룹" 같은 문맥 참조 표현 사용 (42개 중 4개) |
| **논문 특화 질문** | "S61은 어떤 사람들입니까?", "RoB 평가가 어떻게 되었습니까?" 등 논문 내부 코드/구조 참조 → 다른 청크가 더 높은 유사도를 가질 수 있음 |
| **엄격한 판정 기준** | 정확히 같은 chunk_id가 있어야 성공 → 의미적으로 동일한 내용의 다른 청크는 실패로 처리 |

**결론: Hit@5 = 0.095는 시스템이 검색을 못 한다는 의미가 아니라,**  
**18,682개 청크 중 특정 청크 하나를 정확히 맞추기 어렵다는 의미입니다.**

- QA 쌍: "청크 #263을 정확히 가져왔는가?" → 의미적으로 유사한 다른 청크는 실패 처리

### QA 쌍 평가 개선 방향

더 정확한 QA 쌍 기반 평가를 위해서는 다음 개선이 필요합니다:

1. **프롬프트 개선**: "이 텍스트/이 연구" 참조 금지 명시
2. **질문 필터링**: 생성된 질문에서 문맥 의존 표현 자동 탐지 및 제거
3. **사용자 중심 질문**: 실제 사용자가 챗봇에 물어볼 법한 형태로 질문 재구성
4. **중복 청크 제거**: 동일 주제의 청크를 하나만 샘플링하여 중복 최소화

## 5. 멘토 질문 대비 답변

> **"QA 평가를 어떻게 했나요?"**

> 키워드 매칭과 달리 정확한 정답이 있어서 평가 신뢰도가 더 높습니다.
